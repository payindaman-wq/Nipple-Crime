[Heading 1] 0) Tools & access checklist (do this first)
[Heading 2] 0.1 HubSpot permissions (the “don’t get stuck” check)
[normal] If you can’t create/edit properties, lists, or forms, check permissions before troubleshooting anything else.
[normal] Minimum recommended access
[normal] Super Admin (or equivalent permissions to manage properties + marketing tools)
[normal] Access to:
[normal] CRM (contacts)
[normal] Marketing > Forms
[normal] Marketing > Email
[normal] Content / Website Pages (if you’re maintaining the site)
[normal] Commerce (optional; usually Treasurer-owned)
[normal] HubSpot has a full permissions system under Settings → Users & Teams. (HubSpot Knowledge Base)
[normal] Property-level access can also be restricted (so even if you “can edit contacts,” you may not be able to edit certain fields). (HubSpot Knowledge Base)
[Heading 2] 0.2 Sakari integration + basics
[normal] Confirm Sakari is connected to HubSpot (integration setup). When connected, contacts can sync and Sakari features appear in HubSpot records. (Sakari Support Center)
[normal] Know that Sakari pricing is driven by SMS “segments” (longer texts / certain characters cost more). (Sakari Support Center)
[Heading 2] 0.3 Compliance sanity check (keep it simple, but do it)
[normal] Sakari opt-out is carrier keyword–driven (e.g., STOP). You should assume members can text STOP to opt out. (Sakari Support Center)
[normal] Best practice: include “Reply STOP to opt out” periodically (especially on first outreach of the year).
[normal] For marketing email, HubSpot requires unsubscribe/physical address elements through its footer system. (HubSpot Knowledge Base)
[normal] (You don’t need to turn this into a legal thesis—just don’t fight the built-in rules.)
[Heading 1] 1) HubSpot data model (how to think about it)
[Heading 2] 1.1 Contacts are organized around the contact record
[normal] Every person is a contact record
[normal] You’ll use contact properties to store structured answers (e.g., “Interested in camp 2026?”, “Volunteer hours”, “Leadership interest”, “Leadership role”). (HubSpot Knowledge Base)
[normal] You’ll use forms to collect/update those properties at scale. (HubSpot Knowledge Base)
[normal] You’ll use lists/segments (active or static) to target the right people. (HubSpot Knowledge Base)
[Heading 1] 2) Naming conventions (so future you doesn’t hate past you)
[Heading 2] 2.1 Property naming
[normal] Use a predictable prefix so properties group well:
[normal] 2026 - Interested in Camp
[normal] 2026 - Volunteer Time Available
[normal] 2026 - Leadership Interest
[normal] 2026 - Leadership Role (Assigned)
[normal] HubSpot lets you create/edit custom properties and choose field types like dropdowns. (HubSpot Knowledge Base)
[Heading 2] 2.2 List/segment naming
[normal] 2026 - Interest: Yes/Maybe/Unknown
[normal] 2026 - Interest: No
[normal] 2026 - Potential Leaders (Interested, No Role Assigned)
[normal] 2026 - Needs Follow-up (No response yet)
[normal] Lists (segments) can be active (auto-updating) or static (snapshot). (HubSpot Knowledge Base)
[normal] Rule of thumb
[normal] Use Active lists for anything ongoing (“yes/maybe/unknown” should update as responses come in).
[normal] Use Static lists for one-time snapshots (“everyone invited to X at the time we sent it”).
[Heading 1] SOP 1 — Create a contact property (dropdown) in HubSpot
[Heading 2] Purpose
[normal] Create structured fields so survey answers become filterable (and reusable in lists, emails, etc.).
[Heading 2] When to use
[normal] Any time you want a standardized answer set:
[normal] Yes / No / Maybe
[normal] 1–2 hrs/week / 3–4 / 4+
[normal] Which role someone holds
[Heading 2] Steps
[normal] In HubSpot, click Settings (gear icon)
[normal] Go to Data Management → Properties (HubSpot Knowledge Base)
[normal] Choose the object type (usually Contact)
[normal] Click Create property
[normal] Set:
[normal] Property label: 2026 - Interested in Camp
[normal] Field type: Dropdown select
[normal] Options: Yes, Maybe, No
[normal] Save.
[Heading 2] Output
[normal] A property you can add to:
[normal] forms, contact views, and list/segment filters.
[Heading 2] Troubleshooting
[normal] If you can’t see “Create property” or you get errors: check permissions (Super Admin / property access). (HubSpot Knowledge Base)

[Heading 1] SOP 3 — Create a “segment/list” for targeting outreach (HubSpot)
[Heading 2] Purpose
[normal] Turn form results into a clean audience for texting/emailing.

[normal] Create a Contact-based list
[normal] Choose list type:
[normal] Active for continuously updating audiences (recommended for “Yes/Maybe/Unknown”)
[normal] Static for a one-time snapshot (HubSpot Knowledge Base)
[normal] Build filters.

[normal] Goal: everyone except “No”.
[normal] Filters:
[normal] 2026 - Interested in Camp is any of:
[normal] Yes
[normal] Maybe
[normal] (and/or) is unknown/empty (depending on how your portal handles blank)
[normal] Save as an Active list.
[Heading 2] Example B: “Potential leaders (willing + no role assigned)”
[normal] This matches your exact use case.

[normal] 2026 - Leadership Role (Assigned) (dropdown of roles, or single-line text)

[Heading 1] SOP 4 — Send a bulk SMS to a HubSpot list (Sakari Campaign)
[Heading 2] Purpose
[normal] Text the right people with the form link.
[Heading 2] Steps
[normal] In Sakari, go to Campaigns → Create Campaign (Sakari Support Center)
[normal] You’ll move through the standard stages:
[normal] Campaign Details
[normal] Contacts
[normal] Conditions
[normal] Messaging
[normal] Schedule (Sakari Support Center)
[normal] In Contacts, select the synced HubSpot list/segment (your “2026 - Interest: Yes/Maybe/Unknown” list)
[normal] Write message (keep it short; link to the form)
[Heading 3] Recommended SMS template (first outreach of the year)
[normal] NC 2026 check-in: are you coming + can you help? Please fill this out: [FORM LINK]
Reply STOP to opt out.
[normal] (“STOP” is the standard opt-out keyword; Sakari supports carrier opt-out keywords.) (Sakari Support Center)
[Heading 2] Preview cost before sending
[normal] In Sakari scheduled campaigns, you can use the three-dot menu → Preview to see estimated cost. (Sakari Support Center)
Sakari also explains that segment count drives cost. (Sakari Support Center)
[Heading 2] Scheduling
[normal] Schedule at a reasonable hour; avoid sending multiple repeats to non-responders too quickly.
